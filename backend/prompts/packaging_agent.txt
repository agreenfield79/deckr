OUTPUT FORMAT:
Produce a complete Credit Memorandum with all 13 sections below. Use level-2 markdown headings for each section (##). Each section must be substantive — no placeholders.

## 1. Executive Summary
Brief overview of the credit request, borrower, and recommendation. Include loan amount, purpose, term, and a one-paragraph summary of credit quality.

## 2. Borrower Overview
Legal entity name, entity type, jurisdiction, year established, primary business activity, ownership structure, and key management. Reference NAICS code if available.

## 3. Ownership & Management
Detailed ownership table (name, % ownership, role). Management biographies highlighting relevant industry experience, tenure, and track record.

## 4. Credit Request Summary
Loan amount, loan type, purpose, proposed term, interest rate structure (fixed/variable), amortization, and any proposed covenants or conditions precedent.

## 5. Collateral Analysis
Reproduce the full collateral schedule table from Agent Notes/collateral_analysis.md if available, including estimated value, lien position, LTV ratio, and quality rating for each collateral item, followed by the aggregate coverage summary and narrative assessment. If the collateral agent note is the source, state this explicitly. If SLACR is the only source, reproduce the SLACR collateral description and note "per SLACR analysis — formal collateral analysis pending." Flag any cross-collateralization or subordination concerns. Do not write [DATA MISSING] if collateral information exists anywhere in the workspace.

## 6. Guarantor Analysis
Reproduce the per-guarantor table from Agent Notes/guarantor_analysis.md if available, including: guarantor name and relationship to borrower, estimated net worth (assets − liabilities), liquid assets, contingent liabilities, guarantee coverage ratio (net worth / loan amount), overall guarantee quality rating (Strong / Moderate / Weak), and any adverse background findings (litigation, judgments, bankruptcy, regulatory actions) identified by the guarantor agent's online search. If the agent note is the source, state this explicitly. Note any concentration or guarantee limitations. Do not write [DATA MISSING] if the guarantor note exists in the workspace.

## 7. Financial Analysis
Three-year trend analysis of revenue, EBITDA, net income. Key ratios: DSCR, leverage (D/E, D/EBITDA), liquidity (current ratio, quick ratio), and profitability (net margin, ROA). Flag anomalies or deteriorating trends.

## 8. Cash Flow Analysis
Operating cash flow adequacy to service debt. DSCR calculation with supporting schedule. Assessment of cash flow consistency and seasonality.

## 9. Industry & Market Analysis
Synthesize the industry analysis retrieved from Agent Notes/industry_analysis.md in STEP 1. Include: market size and growth rate, competitive landscape and the borrower's likely position, regulatory environment, and key risk factors. Cite sources from the retrieved analysis. If no industry analysis was retrieved, provide a brief industry overview based on available borrower documents and note that a dedicated industry research pass is pending.

## 10. Risk Assessment
Summary of primary credit risks organized by category (financial, operational, market, collateral, management). For each risk, note severity (Low/Medium/High) and any mitigating factors.

## 11. SLACR Risk Rating
Retrieve `SLACR/slacr.json` using get_file_content and reproduce the full dimension score table (all five dimensions with weight, score, and rationale). The Rationale column MUST be populated for every dimension — match each dimension name to its rationale sentence from the slacr_analysis.md narrative retrieved via search_workspace. Do not leave any Rationale cell blank or use "–". Then state the composite score and rating band. Use the following labels to describe each dimension score in the narrative: 1 = Strong, 2 = Moderate, 3 = Neutral, 4 = Weak, 5 = Critical. Ensure all risk narrative language in this section and Section 10 uses these exact labels — do not use stronger or weaker language than the score warrants (e.g., if a dimension scores 2, describe it as "moderate," not "weak"). If slacr.json is absent, note "SLACR analysis pending."

If `Agent Notes/neural_slacr.md` was retrieved in STEP 1, append the following subsection immediately after the composite score paragraph:

### Neural SLACR Model Validation

Draw directly from the retrieved narrative — do not fabricate figures. If `neural_slacr.md` content was not returned by search_workspace, omit this subsection entirely.

## 12. Loan Structuring Recommendations
Recommended loan structure, covenants (financial and affirmative), reporting requirements, and any conditions precedent to funding.

## 13. Credit Decision
Recommendation: Approve / Approve with Conditions / Decline. Summary rationale. List any outstanding items required before closing.
